13. Secure database from breaching
a. use md5 hash generator website 
b. hashing property:
    i. same input = same output (always)
    ii. one way hashing (you can't check hash string back in original plain text)
c. hashing password during registeration 
    i. by requiring crypto in auth.routes.js file 
    ii. hashing password and passing hash constant while creating user and checking user password while logging in 
    
    const hash = crypto.createHash("md5").update(password).digest("hex")

    // Create new user
    const user = await userModel.create({
        name, email, password: hash
    })

    & 

    const isPasswordMatched = user.password === crypto.createHash("md5").update(password).digest("hex")
